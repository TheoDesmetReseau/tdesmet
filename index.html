1. Installation Serveur

Infrastructure

Création d’une VM Ubuntu Server et configuration réseau. Installation des paquets (Apache2, MariaDB, PHP, dépendances) puis configuration d’Apache (vhosts) pour héberger GLPI dans un environnement stable et maintenable.

2. Migration Données

Base de données

Migration de l’ancien serveur vers le nouveau via SCP (fichiers applicatifs) et restauration de la base de données sur MariaDB. Vérification de l’intégrité puis mise à niveau/ajustements via l’assistant GLPI si nécessaire.

3. Résolution d'incidents

Debugging

Stabilisation post-migration : gestion des droits (Apache/GLPI/BDD), corrections de configuration et résolution d’erreurs fonctionnelles. Objectif : assurer une mise en service fiable sans impacter les utilisateurs.

Bilan du projet

Le nouveau serveur GLPI est opérationnel et la gestion de parc est fiabilisée pour une volumétrie de 1000+ équipements. La migration a été réalisée de façon transparente (ancien serveur conservé en parallèle le temps de la bascule). Les bases sont maintenant posées pour une évolution vers un vrai système de tickets (workflow + rôles + attribution + reporting).
